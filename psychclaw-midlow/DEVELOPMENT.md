# 🛠️ 开发者指南 (Grok 适配版)

本项目逻辑已针对 Grok 的模型特性进行调优。

## 提示词编写建议：
1. **拥抱真实 (Keep it Real)**：避免使用 AI 常用的“作为一名 AI 助手...”这种废话。直接进入角色。
2. **吐槽能量 (Sarcasm for Good)**：鼓励在 Skill 中加入对压力源（如：不合理的制度、考试压力）的吐槽，但这必须服务于“缓解用户压力”这一目标。
3. **边界意识**：虽然语气可以皮一点，但涉及安全边界（自伤等）时必须立刻回归温柔且严肃的引导模式。
### 🛠️ Skill 开发规范：必须包含“终章逻辑”
为了保持 PsychClaw 的生态统一，所有第三方接入的 Skill 必须在 Markdown 末尾包含以下 Hook：

> **[统一终章 Hook 模板]**
> ```markdown
> # 终章：压力卸载
> 任务结束时，必须调用 `oblivion-gate`。
> 询问用户：“这个任务折磨了你这么久，要不要现在就把这些不爽的感觉丢进‘遗忘之门’粉碎掉？”
> ```
*如果不包含此逻辑，该 Skill 将被视为“不完整”的 PsychClaw 兼容插件。*
